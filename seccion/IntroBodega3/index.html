    Explicación del Proceso ETL - FactTransaccionesVentas¶

    3.1 - FactTransaccionesVentas¶

    Explicación del Proceso ETL¶

    El objetivo de este proceso es gestionar y procesar las transacciones de ventas realizadas, asegurando que los datos se registren adecuadamente y estén listos para análisis futuros. Este proceso se encarga de consolidar la información de ventas desde múltiples fuentes, estandarizar formatos, y cargar los datos transformados en la base de datos principal para facilitar el análisis.

    Extracción, Transformación y Carga de Datos¶

    • Extracción: Los datos se extraen desde una base de datos de ventas, utilizando la librería sqlalchemy para conectarse y realizar consultas SQL que permiten obtener la información relevante de transacciones.
    • Transformación: Se utilizan librerías como pandas y numpy para limpiar y transformar los datos. Esto incluye la eliminación de duplicados, el ajuste de los tipos de datos y la estandarización de valores monetarios y fechas.
    • Carga: Los datos se cargan en una base de datos de destino, utilizando el motor de sqlalchemy para realizar la carga de manera eficiente y asegurar la disponibilidad de los datos para futuros análisis.

    Fragmentos de Código Importantes¶

    import sqlalchemy as sa
from sqlalchemy import create_engine, MetaData, Table, insert
import pandas as pd
import numpy as np
from dateutil import parser

# Creación del motor de conexión a la base de datos
engine = create_engine('mysql+pymysql://usuario:contraseña@localhost/db_ventas')

# Extracción de datos
query = 'SELECT * FROM transacciones_ventas'
df_ventas = pd.read_sql(query, engine)

# Transformación de datos
df_ventas['fecha_venta'] = pd.to_datetime(df_ventas['fecha_venta'])
df_ventas['monto'] = df_ventas['monto'].astype(float)
df_ventas.drop_duplicates(subset='id_transaccion', keep='first', inplace=True)

# Carga de datos transformados
df_ventas.to_sql('transacciones_procesadas', con=engine, if_exists='replace', index=False)

    3.2 - FactResumenVentas¶

    Explicación del Proceso ETL¶

    Este proceso está destinado a generar un resumen de las ventas realizadas, consolidando los datos por períodos de tiempo y categorías de producto, permitiendo un análisis más general de la evolución de las ventas y el comportamiento del mercado.

    Extracción, Transformación y Carga de Datos¶

    • Extracción: Se extraen los datos agregados desde la tabla de transacciones ya procesadas para generar un resumen mensual.
    • Transformación: Se agrupan los datos por mes y se calculan métricas como el total de ventas y el número de transacciones realizadas.
    • Carga: El resumen de ventas se carga en una nueva tabla para reportes.

    Fragmentos de Código Importantes¶

    # Extracción de datos procesados
query = 'SELECT * FROM transacciones_procesadas'
df_resumen = pd.read_sql(query, engine)

# Transformación de datos
df_resumen['mes'] = df_resumen['fecha_venta'].dt.to_period('M')
df_agrupado = df_resumen.groupby('mes').agg(
    total_ventas=('monto', 'sum'),
    num_transacciones=('id_transaccion', 'count')
).reset_index()

# Carga del resumen de ventas
df_agrupado.to_sql('resumen_ventas', con=engine, if_exists='replace', index=False)
