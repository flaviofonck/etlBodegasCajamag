    Explicación del Proceso ETL - Bloque1¶

    Este documento proporciona una explicación detallada del proceso ETL (Extracción, Transformación y Carga de datos) realizado en el archivo Bloque1. El flujo se encuentra dividido en distintas sesiones:

    • 1.2 - EstandarAfiliados2.0
    • 1.3 - CalendarioMensual
    • 1.4 - Dimensiones
    • 1.5 - FactAfiliacion
    • 1.6 - FactDatosContacto

    Cada una de estas sesiones representa una etapa clave del procesamiento de datos, proporcionando una estructura clara para garantizar la eficiencia y la consistencia de la información tratada. A continuación, se explica cada uno de los componentes.

    1.2 - EstandarAfiliados2.0¶

    Explicación del Proceso ETL¶

    Este módulo se encarga de la estandarización de los datos de los afiliados. La estandarización incluye la limpieza de datos, la transformación a formatos consistentes y la validación para asegurar la calidad de los datos antes de su integración en la base de datos.

    Extracción, Transformación y Carga de Datos¶

    Los datos se extraen de fuentes externas como archivos CSV o tablas SQL. Posteriormente, se transforman aplicando reglas de negocio para garantizar la coherencia. Finalmente, se cargan en una tabla de afiliados estandarizada.

    Fragmentos de Código Importantes¶

    A continuación, se presentan algunos fragmentos de código que son fundamentales para la extracción, transformación y carga de datos en esta sesión:

    import pandas as pd
import sqlalchemy as sa

# Creación de la conexión con la base de datos
engine = sa.create_engine('postgresql://usuario:password@host:puerto/basedatos')

# Extracción de datos desde CSV
df_afiliados = pd.read_csv('afiliados.csv')

# Limpieza y estandarización de datos
df_afiliados['fecha_nacimiento'] = pd.to_datetime(df_afiliados['fecha_nacimiento'], errors='coerce')
df_afiliados['nombre'] = df_afiliados['nombre'].str.upper()

# Carga de datos a la base de datos
df_afiliados.to_sql('afiliados_estandarizados', con=engine, if_exists='replace', index=False)

    1.3 - CalendarioMensual¶

    Explicación del Proceso ETL¶

    El módulo CalendarioMensual se encarga de definir y gestionar las actividades ETL mensuales, asegurando que cada proceso de carga se realice en el momento adecuado. Esto permite una planificación eficiente del flujo de datos.

    Extracción, Transformación y Carga de Datos¶

    Se extraen datos de diferentes períodos para organizarlos en un calendario mensual. Las fechas se utilizan para definir la planificación de la carga, facilitando análisis temporales consistentes.

    Fragmentos de Código Importantes¶

    A continuación, se muestran fragmentos de código para la creación y gestión del calendario mensual:

    import pandas as pd
import datetime

# Definir el rango de fechas para el calendario mensual
fecha_inicio = '2023-01-01'
fecha_fin = '2023-12-31'

# Creación del DataFrame del calendario
df_calendario = pd.date_range(start=fecha_inicio, end=fecha_fin, freq='M').to_frame(index=False, name='fecha')

# Añadir columnas adicionales
df_calendario['mes'] = df_calendario['fecha'].dt.month
df_calendario['año'] = df_calendario['fecha'].dt.year

print(df_calendario)

    1.4 - Dimensiones¶

    Explicación del Proceso ETL¶

    Este módulo se encarga de definir y gestionar las dimensiones del modelo de datos, tales como afiliados, productos, o cualquier otra entidad relevante. Las dimensiones son tablas descriptivas que proporcionan contexto a los datos transaccionales, enriqueciendo el análisis.

    Extracción, Transformación y Carga¶

    Los datos se extraen, se procesan para definir dimensiones consistentes, y luego se cargan para ser utilizados en consultas y análisis posteriores.

    Fragmentos de Código Importantes¶

    A continuación, se presentan algunos fragmentos de código relevantes para la creación de las dimensiones:

    import pandas as pd
import sqlalchemy as sa

# Creación de la conexión con la base de datos
engine = sa.create_engine('postgresql://usuario:password@host:puerto/basedatos')

# Extracción de datos desde la tabla de afiliados
df_afiliados = pd.read_sql('SELECT * FROM afiliados', con=engine)

# Creación de la dimensión afiliados
df_dim_afiliados = df_afiliados[['afiliado_id', 'nombre', 'edad', 'genero']]

# Carga de la dimensión a la base de datos
df_dim_afiliados.to_sql('dim_afiliados', con=engine, if_exists='replace', index=False)

    1.5 - FactAfiliacion¶

    Explicación del Proceso ETL¶

    El módulo FactAfiliacion crea una tabla de hechos que almacena los datos transaccionales relacionados con la afiliación. Esto permite realizar análisis históricos sobre las afiliaciones, tales como el comportamiento de los afiliados a lo largo del tiempo.

    Extracción, Transformación y Carga de Datos¶

    Se extraen los datos de afiliación, se transforman para adaptarse al esquema de la tabla de hechos y se cargan en la base de datos para análisis y generación de reportes.

    Fragmentos de Código Importantes¶

    A continuación, algunos fragmentos de código utilizados para la creación de la tabla de hechos de afiliación:

    import pandas as pd
import sqlalchemy as sa

# Creación de la conexión con la base de datos
engine = sa.create_engine('postgresql://usuario:password@host:puerto/basedatos')

# Extracción de datos transaccionales
df_transacciones = pd.read_sql('SELECT * FROM transacciones_afiliacion', con=engine)

# Transformación de datos para adaptarse al esquema de hechos
df_fact_afiliacion = df_transacciones[['afiliado_id', 'fecha_afiliacion', 'tipo_afiliacion']]
df_fact_afiliacion['mes'] = pd.to_datetime(df_fact_afiliacion['fecha_afiliacion']).dt.month

# Carga de la tabla de hechos
df_fact_afiliacion.to_sql('fact_afiliacion', con=engine, if_exists='replace', index=False)

    1.6 - FactDatosContacto¶

    Explicación del Proceso ETL¶

    El módulo FactDatosContacto gestiona una tabla de hechos que contiene información sobre los datos de contacto de los afiliados. Estos datos incluyen teléfonos, correos electrónicos y direcciones, fundamentales para la comunicación y seguimiento de los afiliados.

    Transformación y Carga¶

    Los datos se extraen, se limpian y se transforman para estandarizar la información de contacto. Luego, se cargan en la base de datos para que puedan ser utilizados en análisis y comunicaciones.

    Fragmentos de Código Importantes¶

    A continuación, se presentan algunos fragmentos de código utilizados para manejar los datos de contacto:

    import pandas as pd
import sqlalchemy as sa

# Creación de la conexión con la base de datos
engine = sa.create_engine('postgresql://usuario:password@host:puerto/basedatos')

# Extracción de datos de contacto
df_contacto = pd.read_csv('datos_contacto.csv')

# Limpieza y estandarización de los datos de contacto
df_contacto['telefono'] = df_contacto['telefono'].str.replace('-', '').str.strip()
df_contacto['email'] = df_contacto['email'].str.lower()

# Carga de los datos de contacto estandarizados
df_contacto.to_sql('fact_datos_contacto', con=engine, if_exists='replace', index=False)
