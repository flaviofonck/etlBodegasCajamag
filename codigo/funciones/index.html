    Introducción¶

    Este documento proporciona una explicación detallada de las funciones incluidas en el archivo Funciones.py, utilizadas en los procesos ETL. A continuación, se incluye una descripción general de las librerías utilizadas, una descripción de cada función, así como un diagrama de procesos y un ejemplo del código fuente principal de cada función.

    Librerías a Utilizar¶

    • pandas: Utilizada para la manipulación y transformación de datos.
    • os: Proporciona funciones para interactuar con el sistema operativo.
    • sqlalchemy: Utilizada para la conexión y manejo de bases de datos SQL.
    • time: Incluida para medir el tiempo de ejecución de ciertas operaciones.
    • datetime: Utilizada para manejar fechas y tiempos.
    • logging: Proporciona capacidades de registro de eventos, útil para el monitoreo de procesos ETL.

    Descripción de Funciones¶

    1. eliminar_tablas_con_llaves_foraneas(tablas)¶

    Descripción¶

    Esta función se encarga de desactivar las restricciones de llaves foráneas, eliminar las tablas en el orden correcto y luego reactivar dichas restricciones. Esto es especialmente útil cuando se necesita eliminar múltiples tablas relacionadas sin causar conflictos debido a restricciones de llaves foráneas.

    • Parámetros:
    • tablas: Lista de nombres de tablas a eliminar en orden específico.

    Diagrama de Proceso¶

    graph TD;
    A[Desactivar Restricciones de Llaves Foráneas] --> B[Eliminar Tablas en Orden]
    B --> C[Reactivar Restricciones de Llaves Foráneas]

    Fragmento de Código Principal¶

    def eliminar_tablas_con_llaves_foraneas(tablas):
    with engine.connect() as connection:
        connection.execute(sa_text("SET FOREIGN_KEY_CHECKS=0"))
        for tabla in tablas:
            connection.execute(sa_text(f"DROP TABLE IF EXISTS {tabla}"))
        connection.execute(sa_text("SET FOREIGN_KEY_CHECKS=1"))

    2. crear_engine_bd(conexion_string)¶

    Descripción¶

    Esta función crea un motor de conexión a la base de datos utilizando la cadena de conexión proporcionada. Este motor es esencial para realizar operaciones de extracción, transformación y carga en la base de datos.

    • Parámetros:
    • conexion_string: Cadena de conexión para conectarse a la base de datos.

    Diagrama de Proceso¶

    A[Recibir Cadena de Conexión] --> B[Crear Motor de Conexión] --> C[Devolver Motor]

    Fragmento de Código Principal¶

    def crear_engine_bd(conexion_string):
    engine = create_engine(conexion_string)
    return engine

    4. extraer_datos(query, engine)¶

    Descripción¶

    Esta función se encarga de extraer datos de la base de datos utilizando una consulta SQL específica. La función devuelve un DataFrame de pandas que contiene los datos extraídos.

    • Parámetros:
    • query: Consulta SQL para extraer los datos.
    • engine: Motor de conexión a la base de datos.

    Diagrama de Proceso¶

    graph TD;
    A[Recibir Query y Motor de Conexión] --> B[Ejecutar Consulta SQL] --> C[Devolver DataFrame]

    Fragmento de Código Principal¶

    def extraer_datos(query, engine):
    with engine.connect() as connection:
        df = pd.read_sql(query, connection)
    return df

    5. transformar_datos(df)¶

    Descripción¶

    Esta función realiza la transformación de los datos contenidos en un DataFrame. Esto puede incluir operaciones como limpieza de datos, estandarización de formatos, y eliminación de valores nulos.

    • Parámetros:
    • df: DataFrame con los datos a transformar.

    Diagrama de Proceso¶

    graph TD;
    A[Recibir DataFrame] --> B[Limpieza de Datos] --> C[Estandarización de Formatos] --> D[Devolver DataFrame Transformado]

    Fragmento de Código Principal¶

    def transformar_datos(df):
    df = df.dropna()
    df['fecha'] = pd.to_datetime(df['fecha'])
    return df

    6. cargar_datos(df, tabla, engine)¶

    Descripción¶

    Esta función se encarga de cargar los datos transformados en la tabla especificada de la base de datos destino.

    • Parámetros:
    • df: DataFrame con los datos a cargar.
    • tabla: Nombre de la tabla destino.
    • engine: Motor de conexión a la base de datos.

    Diagrama de Proceso¶

    graph TD;
    A[Recibir DataFrame, Tabla y Motor] --> B[Cargar Datos en Tabla]

    Fragmento de Código Principal¶

    def cargar_datos(df, tabla, engine):
    with engine.connect() as connection:
        df.to_sql(tabla, con=engine, if_exists='replace', index=False)

    9. guardar_en_dwh(df, tabla, conexion)¶

    Descripción¶

    Esta función guarda los datos en el almacén de datos (DWH), permitiendo la centralización y almacenamiento a largo plazo de los datos procesados para análisis posteriores.

    • Parámetros:
    • df: DataFrame con los datos a almacenar.
    • tabla: Nombre de la tabla destino en el DWH.
    • conexion: Conexión activa al almacén de datos.

    A[Recibir DataFrame y Conexión] --> B[Guardar en Tabla del DWH]

    Fragmento de Código Principal¶

    df.to_sql(tabla, con=conexion, if_exists='append', index=False)

    • Parámetros:
    • tablas: Lista de tablas a cargar.
    • conexion: Conexión a la base de datos.

    Diagrama de Proceso¶

    A[Seleccionar Tabla] --> B[Iniciar Temporizador] --> C[Cargar Tabla] --> D[Detener Temporizador] --> E[Registrar Tiempo]

    Fragmento de Código Principal¶

    def segundos_cargar_tablas(tablas, conexion):
    for tabla in tablas:
        start_time = time.time()
        # Código para cargar la tabla
        end_time = time.time()
        print(f"Tiempo para cargar {tabla}: {end_time - start_time} segundos")
