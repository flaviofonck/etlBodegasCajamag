    Definición de consultas SQL con columnas requeridas¶

    Se definen consultas SQL en el diccionario qr_structure para extraer los datos de las tablas BD_Datos_Estudiantes y BD_Datos_Acudientes. Se asegura que las columnas necesarias, como id, fecnac, sexo, coddoc, numdocumento, prinom, segnom, priape, y segape, estén presentes en los resultados. Se utiliza la función create_engine() para conectar a la base de datos DWH. La conexión exitosa se registra en el log.

    # Definir las consultas SQL para cargar datos y asegurarse de crear la estructura con los campos requeridos
required_columns = ['id', 'fecnac', 'sexo', 'coddoc', 'numdocumento', 'prinom', 'segnom', 'priape', 'segape']
qr_structure = {
    "colegio12": '''
    SELECT
        CONCAT(coddoc, numdoc) AS id,
        fecnac,
        codsex AS sexo,
        coddoc,
        numdoc AS numdocumento,
        prinom,
        segnom,
        priape,
        segape
    FROM BD_Datos_Estudiantes''',
    "colegio13": '''
    SELECT
        CONCAT(coddoc, numdoc) AS id,
        NULL AS fecnac,
        codsex AS sexo,
        coddoc,
        numdoc AS numdocumento,
        prinom,
        segnom,
        priape,
        segape
    FROM BD_Datos_Acudientes'''
}

# Conectar a la base de datos DWH
motor = create_engine(Conexion_dwh())
logger.info('CONEXIÓN A BASE DE DATOS DWH ESTABLECIDA')

    Cargar y estructurar tablas con las columnas requeridas¶

    Se cargan las tablas colegio12 y colegio13 desde la base de datos, garantizando que todas las columnas requeridas estén presentes en los dataframes resultantes. Si alguna columna falta, se crea con el valor "SinDato". Posteriormente, se concatenan ambas tablas en un solo dataframe df_colegio_combined. Los eventos de carga y concatenación se registran en el log.

    # Cargar las tablas en DataFrames y asegurar la estructura requerida
df_structure = {}
for key, query in qr_structure.items():
    try:
        with motor.begin() as conn:
            logger.info(f'CARGANDO TABLA: {key}')
            df = pd.read_sql_query(sa.text(query), conn)

            # Asegurarse de que las columnas requeridas estén presentes en la estructura
            for column in required_columns:
                if column not in df.columns:
                    df[column] = "SinDato"

            df_structure[key] = df[required_columns]
            logger.info(f'TABLA {key} CARGADA Y ESTRUCTURADA')
    except Exception as e:
        logger.error(f'Error al cargar la tabla {key}: {str(e)}')

# Concatenar las tablas de colegio12 y colegio13
df_colegio_combined = pd.concat([df_structure['colegio12'], df_structure['colegio13']], ignore_index=True)
logger.info('TABLAS colegio12 Y colegio13 COMBINADAS')

    Guardar registros faltantes en la tabla DimDatosFijos¶

    Se conectan los registros faltantes en la tabla BD_DimDatosFijos en el DWH y se almacenan los registros del dataframe df_missing_in_dwh en la base de datos utilizando el modo append. El proceso de almacenamiento se registra en el log, y en caso de error, se captura y se registra el mensaje correspondiente.

    # Conectar a la base de datos DWH
motor3 = create_engine(Conexion_dwh())
# Guardar los registros faltantes en la tabla DimDatosFijos
almacenamiento_time = time.time()
try:
    df_missing_in_dwh.to_sql(name='BD_Dim_datosfijos', con=motor3, if_exists='append', index=False)
    logger.info(f'ALMACENAMIENTO DE REGISTROS FALTANTES EN DimDatosFijos --- {time.time() - almacenamiento_time:.2f} seconds ---')
except Exception as e:
    logger.error(f'Error al almacenar registros en DimDatosFijos: {str(e)}')
